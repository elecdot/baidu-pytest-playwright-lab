# Docs

This directory contains project-level documentation for the Baidu Map Playwright
automation lab. Keep these files aligned with the executable test suite and the
commands documented in the repository root.

## Documentation Index

- [design_book.md](design_book.md) - The primary experiment design document,
  covering goals, scope, test strategy, test data, test cases, evidence, risks,
  metrics, and report structure.

## Maintenance Rules

- Update `design_book.md` whenever test scope, project commands, dependencies,
  or artifact locations change.
- Prefer concrete project commands over generic tool examples.
- Keep external links clean and point them at canonical documentation.
- Keep implementation details that are likely to change in directory-specific
  README files or source comments instead of duplicating them across documents.
